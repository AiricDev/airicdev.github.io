Airic converts playbooks, SOPs, expert workflows, and team practices into **document-defined agent packs** you can run, reflect on, and improve.

## The core loop

1. **Source** — methodology lives in documents (playbooks, SOPs, courses, expert knowledge)
2. **Pack** — Packsmith extracts it into an Airic extension pack
3. **Work** — agents execute guided processes against task documents
4. **Reflection** — agents review the run and surface friction or missing rules
5. **Patch** — improvements become human-reviewable Markdown diffs

Everything stays in editable Markdown files inside your workspace — not hidden prompts or vendor black boxes.

## What makes Airic different

| Traditional agents | Airic |
| --- | --- |
| Hidden prompts | Editable Markdown behavior |
| One-off chats | Durable executable documents |
| Silent memory | Human-approved spec patches |

See [Getting Started](/docs/getting-started/) to set up your first `.airic/` workspace, or explore [Packsmith](/docs/packsmith/) to build a pack from source material.
